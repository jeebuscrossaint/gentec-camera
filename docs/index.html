What is this?

Controlling professional IR beam profiling hardware typically requires C# or C++ integration through vendor SDKs. This project bridges that gap — wrapping the Gentec BeamageSDK.dll via pythonnet to expose a clean, ergonomic Python API. Researchers at the UCF CREOL Astrophotonics Lab can now capture and analyze 2048×2048 IR beam profiles, extract beam statistics, and export publication-ready data without leaving Python.

Features

⚡

Simple Python API

connect(), capture(), get_beam_stats() — one-line calls over a full .NET SDK.

📡

Real-Time Beam Analysis

4-sigma beam diameters (X & Y), centroid tracking, FPS, and per-frame statistics streamed live from hardware.

🔭

FITS Export

Full astronomical FITS headers: timestamp, exposure, beam stats, wavelength range, instrument metadata — ready for publication.

🖼️

Smart Visualization

Full-sensor view + auto-detected peak zoom with crosshair overlay. Percentile-scaled hot colormap, saved to PNG automatically.

🧵

Event-Driven Architecture

Asynchronous .NET event callbacks, thread-safe image buffering, and timeout-protected capture for robust operation.

🖥️

Full CLI

Argparse-powered interface with auto/manual exposure, multi-frame capture, verbosity levels, and serial-number camera selection.

Usage

Auto-exposure, 10 frames
python main.py --auto-exposure --frames 10
Manual exposure, custom output
python main.py --exposure 50.5 --output capture.fits
Target specific camera by serial
python main.py --serial 228451 --auto-exposure
Quiet mode, no visualization
python main.py --exposure 100 --frames 5 --no-plot
